## Design Context

### Users
Recruiters, hiring managers, and other developers evaluating Mathieu Schrurs' work — typically a quick scan rather than a deep read, roughly equal split between desktop and mobile viewing. They're judging signal of competence and taste in a short visit, not consuming this as a resume line-by-line.

### Brand Personality
Quietly confident, systems-thinker, craftsman. Drawn directly from the site's own About copy: "does its job quietly... well-structured, easy to reason about, and built to last beyond the first release." Background outside code (music publishing, teaching, basketball coaching) signals discipline and an unconventional path, not a flashy one. The tone is understated confidence with real craft underneath — not hype, not loud.

### Aesthetic Direction
Current site identity: purple accent (`--accent-color`, `#a78bfa` dark / `#8b5cf6` light), SF Mono for labels/metadata, Calibre for body text, numbered section headings ("01.", "02."...), light/dark theme support via `ThemeModeContext`. This identity is not sacred — open to evolving it where a stronger direction demands, starting with the Experience/"Where I've Worked" section, which was flagged as too generic (the well-known alternating-card/circle-node dev-portfolio template) with no motion.

Anti-references (implicit, from impeccable's own guidelines, confirmed applicable here): side-stripe accent borders on cards, generic box-shadow-lift hover glow, gradient text. The current `ExperienceCard`'s `border-left: 1px solid var(--accent-color)` is exactly the banned side-stripe pattern and should not be reintroduced in any redesign.

**Reference (added after first Experience-section pass)**: https://www.roshan-sahu.com/ — cited for layout/scroll experience, not palette. That site uses full-bleed, edge-to-edge project panels (each project its own large "scene" with generous negative space used deliberately, oversized display typography for the title, GSAP-driven scroll reveals) rather than a narrow constrained column. Borrow the STRUCTURE — full width usage, two-zone per-entry layout, oversized type as a moment, scroll-triggered per-entry reveal — not the loud saturated color-blocking, which would contradict the quiet-confidence brand personality above. The first Experience-section pass (a single narrow 660px column, left-aligned, with an ink-line/ignite motif) read as "sitting on the left, not covering the page" on large desktop screens — confirmed direction is to keep the ink-line as a secondary connecting thread while rebuilding entries as full-width scenes.

**Mono-as-body is deliberate (confirmed 2026-07-12):** body copy site-wide stays SF Mono by explicit owner choice — it is the site's signature, not an oversight. Calibre is reserved for display moments (hero name, Experience company names, section headings, the Contact statement). Do not propose moving paragraphs to Calibre again; de-template through composition and scale instead.

**Work cards are terminal windows (2026-07-12):** dots title bar + slugified filename, `$`-prompt description with hover-blink cursor, tags as `--flags`. Extend this language rather than replacing it.

### Design Principles
- Understated over flashy: motion and color should read as precise, not decorative for its own sake.
- Extend existing motifs rather than invent unrelated ones (e.g. the site's existing "signature draw" hover animation on the logo is a natural seed for bigger scroll-driven line-drawing motion elsewhere).
- Single coherent layout across breakpoints beats device-specific markup forks where the content doesn't actually require it.
- Respect `prefers-reduced-motion` (via the existing `usePrefersReducedMotion` hook) for any scroll-linked or ambient animation.
- Avoid recognized generic/AI-slop patterns on sight: side-stripe borders, gradient text, box-shadow-glow-on-hover as the default "polish" move.
